## 二、内容清单（需准备的素材）

| 可下载简历   | PDF 文件                             | 放在 `public/resume.pdf`                |

## 四、配色方案（三选一）

### 🎨 方案一：深邃科技蓝
| 用途                       | 色值      | 备注         |
| -------------------------- | --------- | ------------ |
| 主色（导航、页脚、大标题） | `#0A1929` | 深海军蓝     |
| 辅色（按钮、图标、链接）   | `#1E3A6F` | 宝蓝         |
| 背景色（页面主体）         | `#F8FAFC` | 浅灰白       |
| 卡片背景                   | `#FFFFFF` | 白色，带阴影 |
| 主文字                     | `#1E293B` | 深灰         |
| 次要文字                   | `#64748B` | 中灰         |
| 强调色（悬停、重要按钮）   | `#3B82F6` | 亮蓝         |
| 点缀色（成功提示）         | `#10B981` | 翠绿         |

### 🎨 方案二：柔和极简风
| 用途                     | 色值      | 备注   |
| ------------------------ | --------- | ------ |
| 主色（大面积背景）       | `#FFFFFF` | 纯白   |
| 辅色（卡片背景、分割线） | `#F3F4F6` | 浅灰   |
| 整体背景                 | `#F9FAFB` | 极浅灰 |
| 主文字                   | `#111827` | 近乎黑 |
| 次要文字                 | `#6B7280` | 中灰   |
| 强调色（链接、按钮）     | `#8B5CF6` | 淡紫   |
| 点缀色（标签、小点缀）   | `#EC4899` | 粉红   |

### 🎨 方案三：活力渐变风
| 用途             | 色值                                        | 备注                 |

## 六、开发准备与依赖

### 6.1 初始化项目
```bash
npm create vue@latest personal-website
cd personal-website
npm install
```

### 6.2 核心依赖
| 依赖                   | 用途                      |
| ---------------------- | ------------------------- |
| `vue-router`           | 路由管理                  |
| `pinia`                | 状态管理（如需主题切换）  |
| `@vueuse/head`         | SEO 动态标题/元信息       |
| `vite-plugin-markdown` | 支持导入 Markdown 为组件  |
| `gray-matter`          | 解析 Markdown frontmatter |
| `sass`                 | 使用 SCSS（可选）         |
| `vue3-carousel` 等     | 轮播组件（如需）          |

### 6.3 Vite 配置示例（`vite.config.js`）
```js
import { fileURLToPath, URL } from 'node:url'
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import Markdown from 'vite-plugin-md'

export default defineConfig({
  plugins: [
    vue({
      include: [/\.vue$/, /\.md$/] // 允许导入 .md 文件作为 Vue 组件
    }),
    Markdown()
  ],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  }
})
```

## 七、部署建议

### 7.1 构建
```bash
npm run build
```
生成 `dist` 目录。

### 7.2 部署平台
- **Vercel**：连接 GitHub 仓库，自动部署，支持路由回退。
- **Netlify**：类似 Vercel，配置简单。
- **GitHub Pages**：免费，需配置 `404.html` 或使用 hash 模式。

### 7.3 路由模式
使用 `createWebHistory()` 需服务器支持回退到 `index.html`。Vercel/Netlify 默认支持。
